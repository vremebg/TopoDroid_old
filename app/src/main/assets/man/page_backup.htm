Sketch Reload

This dialog is opened by the Reload menu of the Sketch window.

TopoDroid maintains backup copies of sketch files, from which the sketch can be recovered.
TopoDroid keeps a set of four backup copies for each sketch file. These are named with suffixes "bck", "bck0", "bck1", and "bck2" and are stored in the tdr subdirectory (binary sketch files). Whenever a sketch is saved the backup copies are rotated (eg, "bck1" is renamed "bck2", and so on; "bck" is renamed "bck0"), the old "tdr" file is renamed with suffix "bck", and the sketch is saved to a new "tdr" file.

TopoDroid saves a sketch whenever it is modified (eg, a new item is added, or part of the sketch is erased) while it is open. Only one backup is used in this case: the old saved copy being renamed with suffix "bck".

When you switch between plan and profile view, or to and from a cross-section, and when you close the sketch window, the open view is saved.

The dialog shows the list of the sketch backups with

  • how long ago it was written, with suffixes for minutes (m), hours (h), days (d), months (M) and years (Y)
  • size of the file, in square brackets, in bytes
  • filename

To recover a sketch you choose which copy to load, and tap the OK button to confirm.
The Back key closes the dialog without reloading the sketch.
